# Why V120 is not final

V120 is preserved as a detail-verified pipeline checkpoint, not final mentor evidence. The decisive issue is that visual proof still leaned on high-density point-cloud boards, local point closeups, and self-reported JSON gates. Code audit confirms config-specific generation and scoring terms, including `weighted_pick(..., replace=True)`, true-specific contrast amplification, `detail_bonus`, and `control_penalty`. The new route therefore requires camera/RGB/mask projection evidence, config-neutral scoring, refined detail masks, hard controls, and a Yuque-style report before any mentor-ready statement.
